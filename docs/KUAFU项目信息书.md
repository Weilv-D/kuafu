# 夸父 KUAFU — 桌面双轮腿机器人

面向强化学习的桌面级双轮腿验证平台。采用"大脑 + 小脑"分层架构，在 1500 元预算内整合轮式平衡、腿式姿态变换与视觉导航闭环。

| 约束 | 目标值 |
|------|--------|
| 整机重量 | ≤ 2.5 kg（核算 **2.21 kg**，含充电宝） |
| 底盘宽 | ~231 mm（含胎），侧板间距 196 mm |
| 驻留 / 最大高度 | 147 mm / 296 mm（机身顶） |
| 预算 | ≤ 1500 元 |
| 控制周期 | 姿态估计 **1 kHz**（IMU）；平衡控制 **~250 Hz**（RS485 电机环）；决策 **30–50 Hz**（Pi） |

### 任务范围与验收标准

| 任务 | 量化指标 | 关键依赖 |
|------|---------|--------|
| **轮行平衡**（平地） | 连续平衡 ≥10 min，3 次侧向轻推（~2 N·s）恢复时间 ≤2 s | STM32 平衡环 + 扭矩裕度 |
| **腿姿态变换** | 左右地形差 ±25 mm 内保持机身水平，切换时间 ≤0.5 s，静态误差 ≤2° | 腿垂向行程 + 承重面自锁 |
| **台阶攀爬** | 30 mm 单级台阶，~1 s/级，≥20 次成功率 ≥80%，爬阶后 2 s 内恢复平衡 | 舵机转速 + 五杆行程 |
| **视觉导航** | D435i 避障，目标点 ≤2 m，偏差 ≤10 cm，≥10 次成功率 ≥80% | Pi RL 策略 + VIO |

机构物理补偿能力在驻留态可达 ±45 mm，中段（D₀=100–145）峰值 ±118 mm，对 ±25 mm 需求有 4× 以上富余。

## 一、系统架构

| 层 | 硬件 | 职责 |
|----|------|------|
| 大脑 | 树莓派 5（8GB） | ROS 2 Jazzy、D435i VIO/SLAM、RL 策略推理（ONNX） |
| 小脑 | STM32F407ZGT6 | 姿态估计 1 kHz Mahony；电机环 ~250 Hz；底层平衡控制器 |
| 执行 | DDSM315 ×2 + ST3215 C018 ×4 | 轮毂直驱 + 对称并联五杆髋关节（每腿 2 DOF，连杆不交叉） |
| 感知 | BMI088 + D435i | 本体姿态 + 环境深度 |

控制范式以强化学习为目标，具体路线（残差 RL / 端到端 / 分层 / 教师-学生 / 模仿+RL）在下阶段确定。硬件按路线无关的基础设施设计，预留算力与接口兼容各 RL 范式。当前建议 M5 阶段以残差 RL 起步（安全、样本高效），M6 视觉导航升级分层 RL，底层平衡技能复用。

安全关键功能全部下沉 STM32 硬实时层，不依赖树莓派。驻留态零力矩自锁是控制降维的核心机制：D₀=58 为最低姿态，曲柄贴合侧板承重面，腿成纯被动结构，平衡环只控 2DOF 轮（经典倒立摆），LQR 降为 2 阶，RL 状态空间最小，sim-to-real gap 收窄。腿仅在姿态变换和爬阶时参与控制。自锁为载荷自紧负反馈——载荷越大曲柄压得越紧，无需主动控制维持。

Pi 与 STM32 之间通过 UART 3.3V TTL 通信，波特率 921600。帧格式：起始 0xA5 + 命令字 1B + 载荷 N B + CRC-8/MAXIM + 结束 0x5A。STM32 上行姿态/状态 ≥200 Hz，Pi 下行控制指令 30–50 Hz。Pi TX→STM32 RX 经 1.8k/3.3k 分压。心跳超时 200 ms 触发 STM32 自主站立保护。

## 二、硬件

### 2.1 执行层

**轮毂电机 DDSM315**（微雪）

| 参数 | 规格 |
|------|------|
| 总线 / 协议 | RS485，**115200 固定**，10 字节帧，CRC-8/MAXIM |
| 电压 | 额定 18V DC / 12–24V |
| 扭矩 | 额定 **0.55 Nm**，堵转 **1.1 Nm** |
| 转速 | 空载 315±10 rpm，额定 200 rpm |
| 重量 | 349±5 g |
| 安装 | 4× M2.1×6 mm，Ø15.2 mm 节圆 |

**髋关节舵机 ST3215 C018**（飞特）

| 参数 | 规格 |
|------|------|
| 扭矩 | 堵转 **2.94 Nm @12V**（C018，1:345 金属齿）；连续扭矩按热特性推算 ~1 Nm |
| 电压 | 6–12.6V |
| 总线 | TTL 半双工，串联，**1 Mbps** |
| 转速 | 0.222 s/60°（45 rpm）@12V 空载 |
| 编码器 | 12 位磁编码器，分辨率 0.088°/LSB |
| 尺寸 / 重量 | 45.22×35×24.72 mm / ~60 g |

### 2.2 感知层

**IMU BMI088**：加速度 ±24g、陀螺 ±2000°/s、SPI ≤10 MHz、ODR 1.6 kHz / 2 kHz。

**深度相机 D435i**：USB 3.0，ROS 2 realsense2_camera。

### 2.3 能源

**5S LiPo 2200 mAh 95C**：标称 18.5V / 满电 21.0V，~285 g，尺寸 137×43×37 mm。

| 部件 | 规格 |
|------|------|
| 分电板 | 纯铜无源，2 oz |
| 舵机降压 | 同步整流 buck（12V，持续最大 20A）|
| STM32 降压 | MP1584 |
| 电容 | 470µF/35V + 0.1µF 去耦 |
| 树莓派供电 | 充电宝独立 5V/3A+，与主电池仅 GND 单点跨接 |

供电时序：先主电池（STM32 就绪）→ 后充电宝（Pi 启动）；下电先 Pi `sudo shutdown` → 后主电池断开，防止 Pi I/O 向未上电的 STM32 灌电流。

**供电独立性约束（安全关键）**：STM32 必须由主电池经 MP1584 独立供电，**禁止经树莓派 USB 口反向取电**。STM32 板载 USB 口的 VBUS 直连板载 5V 轨，插上即取电——一旦 Pi 崩溃或 USB 总线复位，STM32 会跟随掉电，§七"树莓派崩溃→STM32 独立维持站立"的保护机制即告失效。板载 USB 口仅用于离线烧录/调试，运行时不插。

### 2.4 通信

SP3485（3.3V RS485，DDSM315）；飞特 URT-1（ST3215 ID / 零位设定）。

## 三、整机包络

| 尺寸 | 值 |
|---|---|
| 整机包络 | 高 147（驻留）/ 296（最高）× 长 ~240 × 宽 231（含胎）mm |
| 侧板间距 | 196 mm（= 轮距） |
| 轮径 | Ø78.16（r=39.08），胎宽 34.8 |
| 整机重量 | ~2.21 kg（含充电宝）/ ~2.05 kg（系留） |

FDM 制造。承力件（大腿、小腿、轮端叉口）用 PETG（抗摔韧性好、耐热 ~80°C），非承力件（侧板、甲板、支架）用 PLA。驻留态舵机零力矩不发热。瞬态脉冲动作下最坏工况（D₀=145，τ=1.92 Nm）单次脉冲温升仅 ~0.17 K，裕度极大；频繁连续脉冲（≥0.5 Hz）由首件实测确认。关节座默认加散热孔与薄铝隔离。

驻留态零力矩自锁：D₀=58 为最低姿态，曲柄贴合侧板内侧承重面，载荷经曲柄杆身→承重面→侧板→机架刚性传递，绕过舵机轴，舵机仅通保持电流 ~50–80 mA/台。详细结构实现见《结构设计》。

## 四、可行性核算

### 4.1 重量预算

| 项 | 重量 |
|----|------|
| DDSM315 ×2 | 698 g |
| ST3215 C018 ×4 | 240 g |
| 5S 2200 mAh 电池 | 285 g |
| 舵机 buck（20A）| 140 g |
| 树莓派 5 + NVMe HAT + 风扇 | 85 g |
| D435i | 70 g |
| STM32 + 电源（含 BMI088）| 95 g |
| 分电 + SP3485 | 22 g |
| 充电宝（Pi 独立供电） | 150 g |
| 线材 + 连接器 | 60 g |
| PLA 结构件 | 360 g |
| **合计** | **~2205 g（≈ 2.21 kg）** |

### 4.2 平衡扭矩

倒立摆模型，M=2.21 kg，r=0.0391 m。每轮准静态保持扭矩 τ = M·g·sinθ·r/2。

| 倾角 θ | 每轮 τ | 占额定 0.55 Nm |
|--------|--------|----------------|
| 10° | 0.073 Nm | 13% |
| 20° | 0.144 Nm | 26% |
| 30° | 0.211 Nm | 38% |

扭矩富余量大。瓶颈是 RS485 ~250 Hz 控制环带宽与轮速（额定 200 rpm → 轮缘 0.82 m/s）。

### 4.3 运动能力

舵机转速 0.222 s/60° 为空载值，弹道跳跃不可行。运动能力定义为腿部姿态变换与慢速台阶攀爬。真正跳跃需准直驱（QDD）BLDC 关节。

### 4.4 舵机保持扭矩

驻留态（D₀=58）：曲柄贴合承重面，载荷绕过舵机轴，舵机零力矩。

瞬态（D₀>58，曲柄离开承重面）：对称并联五杆构型，单曲柄扭矩 0.51–1.92 Nm（静力平衡法）。D₀≤70 在连续安全 1 Nm 内；D₀>70 超连续安全但低于堵转 2.94 Nm，短时脉冲可承，动作后须立即回锁。

COM 高：含充电宝 ≈71.1 mm，系留 ≈67.4 mm（驻留态 D₀=58）。详见《结构设计》§六。

### 4.5 功耗与续航

5S 2200 mAh，18.5V 标称。测试条件：室内平整硬地，0.3 m/s 匀速，放电终止 3.3 V/节。

| 工况 | 主电池功耗 | 电流(~18.5V) | 续航 |
|------|-----------|--------------|------|
| 驻留站立 | ~16 W | 0.9 A | ~150 min |
| 轮行平衡 | ~51 W | 2.8 A | ~47 min |
| 瞬态活动腿 | ~66 W | 3.6 A | ~36 min |

自带充电宝供 Pi 时续航 −10%。

| 层 | 选型 |
|----|------|
| OS | Ubuntu Server 24.04.4 LTS (arm64 + raspi) |

## 七、失效模式与保护机制

| 失效场景 | 检测手段 | 保护动作 | 层级 |
|---------|---------|---------|------|
| 舵机过热 | ST3215 内置温度反馈 | 超阈值释放扭矩 → 回承重面锁止 | 固件 |
| LiPo 过放 | 分电板电压采样 | 单节 <3.3V 报警 → <3.0V 断电 | 固件+硬件 |
| 通信中断 | STM32 看门狗 | 平衡环切 PD 兜底，腿回锁，缓减速 | 固件 |
| 树莓派崩溃 | STM32 心跳检测 | STM32 独立维持站立 | 架构 |
| 大倾角摔倒 | IMU 姿态判断 | 舵机卸载 + 电机停转 | 固件 |
| 跌落损伤 | — | PLA 结构 + 轮胎缓冲；关节 M3 螺栓可换 | 结构 |

## 八、开发里程碑

| 阶段 | 目标 | 验收标准 |
|------|------|---------|
| **M0 硬件点亮** | 执行器/传感器单独可控 | 电机阶跃响应 ≤100 ms、舵机定位 ±1°、IMU 噪声密度 ≤2%、D435i 30 fps |
| **M1 开环轮行** | 电机开环驱动 | 手持机身前后移动 ≥0.3 m/s、转向 ±45°/s |
| **M2 闭环平衡** | STM32 PD/LQR 自动平衡 | 平地 ≥10 min，中间 3 次侧向轻推（~2 N·s），每次恢复 ≤2 s |
| **M3 腿姿态变换** | 舵机控制 drop | ±25 mm 高低差保持水平（误差 ≤2°），切换 ≤0.5 s |
| **M4 台阶攀爬** | 五杆爬阶 + 平衡恢复 | 30 mm 台阶，~1 s/级，≥20 次成功率 ≥80% |
| **M5 RL 验证** | 仿真训练→部署 | 残差 RL 优于纯 PD 基线（恢复时间缩短 ≥15% 或扰动承受提升 ≥20%） |
| **M6 视觉导航** | D435i + RL | 避障到达 ≤2 m，偏差 ≤10 cm，≥10 次成功率 ≥80% |

通用测试条件：室内平整硬地，电池 SOC ≥50%，环境温度 20–30°C。
