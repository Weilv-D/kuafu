## 零、决策记录 (ADR)

> 本节是所有技术决策的记录。每条决策记录约束、选项、依据，后续文档与代码以此为准。

### ADR-1 仿真器：MuJoCo MJX（而非 Isaac Lab / Genesis）

- **约束**：训练机为 Windows + WSL2 Ubuntu + NVIDIA 独显。
- **否决 Isaac Lab**：Isaac Sim 硬依赖 Vulkan `ray_tracing` 扩展创建 GPU 设备，WSL2 的 GPU 经 D3D12 桥接暴露，该扩展缺失。NVIDIA 官方明确不支持 WSL2。RTX 4090 实战仍报 `No device could be created`（[IsaacLab Discussion #3769](https://github.com/isaac-sim/IsaacLab/discussions/3769)）。headless 模式同样不可用——`gpu.foundation.plugin` 仍需 Vulkan。这不是配置问题，是不可绕过的架构限制。
- **否决 Genesis**：物理引擎用 Taichi/CUDA，WSL2 上可行，但其闭链(closed-loop) equality constraint 2024-12 才加入，官方自承"closed loop + ball joint 有 bug"（[Issue #22](https://github.com/Genesis-Embodied-AI/genesis-world/issues/22)）。本项目五杆髋是精度敏感的闭链机构，约束求解稳定性是硬伤。宣传"比 MJX 快 10-80x"经 [MuJoCo #2303](https://github.com/google-deepmind/mujoco/discussions/2303) 澄清是针对 Isaac 而非 MJX，独立 benchmark 证实真实吞吐远低。无内置 ONNX 部署栈，无公开硬件部署案例。
- **选定 MuJoCo MJX**：JAX 后端，WSL2 + CUDA 天然友好，无渲染依赖。单 4090 ~65万步/s。闭链 equality constraint 是 MuJoCo 传统强项，工业级稳定。[MuJoCo Playground](https://playground.mujoco.org/) 提供 sim-to-real 完整栈（Go1/G1 真实 zero-shot 部署，84-93% 成功率），含 ONNX 导出。同一 `.xml` 模型可同时跑 MJX(GPU 训练) 与原生 MuJoCo(CPU 可视化/部署验证)，sim-to-real 对拍内置。
- **结论**：MuJoCo MJX 为主仿真器，原生 MuJoCo 作可视化与 sim-to-real 对拍。无需第二仿真器。

### ADR-2 RL 范式：残差 RL（M5）→ Teacher-Student 蒸馏 + RMA（M6）

- **约束**：效果上限优先（非保守渐进）；充分利用硬件全量回传与扭矩范围；安全兜底不依赖树莓派。
- **三轴安全兜底分解**（物理标定驱动, 见 verify_model 11→13 项）：
  - **pitch（不稳定）**：基层 LQR（K 已验证, 0.1s 恢复 5°）→ 安全
  - **roll（条件不稳定）**：基层腿 PD（左右 D0 差, 0.1s 恢复 3°）→ 安全
  - **yaw（任务 DOF, 非安全）**：基层仅条件阻尼（\|ωz\|<0.3 时抑制低频漂移; 大 ωz 关闭——同轴轮 IP 原地旋转耦合 pitch 不稳定, 物理标定确认）; yaw 命令跟踪交 RL 差速残差
  - RL 挂掉 → pitch+roll 安全, yaw 低频不漂移; 高频 yaw 漂移属任务级, 非安全关键
- **选残差 RL 为主路线**：RL 输出叠加在 STM32 三轴基层之上的增量。平衡态残差≈0（驻留态自锁下基层已足够）是涌现特性而非限制；越障/调平/扰动时残差激活。策略挂掉底层仍保 pitch+roll → 满足安全硬约束。
- **残差"激进"而非保守**：轮输出**力矩残差** Δτ（含 yaw 差速, 利用 DDSM 真力矩环），腿输出**位置残差** Δq（叠加在 D0 对称+roll PD 基层之上, 适配 ST3215 位置控制）。动作空间由硬件实际能力决定, 见 ADR-3。
- **M2+轻量 M4 轮腿越障定位**：机构足端=轮（非足底）, 无传统 swing/stance 步态; 单策略 + 接触感知让策略自学"何时伸腿"（Swiss-Mile ANYmal-wheel 范式）。M2=轮滚+腿伸缩适应地形/调平（主）; M4=低速小障短暂轻抬轮缘跨障（辅, 由 roll PD + contact_asymmetry_penalty 限制为短暂）。
- **M6 升级 Teacher-Student + RMA**：teacher actor 以 proprio(148)+z(9)=157 为条件; student adapter 在线预测 z; DAgger 蒸馏。此为 ETH ANYmal 走野外的标准方法论（[Miki et al. Science Robotics 2022](https://leggedrobotics.github.io/rl-perceptiveloco/)）。
- **否决 MoE**：任务连续可插值（轮滚/伸腿同接触拓扑）, 离散专家切换不稳定风险 > 收益。优先级: 单策略 > skill-conditioning > hierarchical > MoE。
- **否决端到端 RL**：轮+腿异构执行器全靠 sim 建模, sim-to-real gap 大; 抛弃已验证的三轴基层, 安全边际丧失。
- **否决分层 RL 为主路线**：技能切换瞬间稳定性、接口设计复杂度高。作为 M6 的演进备选保留, 关键接口预留兼容。

### ADR-3 动作空间：混合（轮力矩残差 + 腿位置目标）

执行器实际控制能力（三重核实：飞特协议手册 + python-st3215 寄存器表 + 微雪 wiki）：

| | 轮 DDSM315 | 腿 ST3215 |
|---|---|---|
| 纯力矩控制 | **✅ 能**。电流环模式直接给 q 轴力矩电流 ±8A，τ=0.75·I N·m，闭环 | **❌ 不能**。无 Goal-Torque/Goal-Current 寄存器；Torque Limit 仅限上限。mode2 PWM 开环是电压≈占空比，非电流闭环 |
| 协议族 | 微雪 RS485 私有 | **STS 协议族**（非 SCS） |
| RL action | **力矩残差 Δτ**（叠加在 LQR 输出上） | **位置目标 q_goal**（+ goal_speed/accel 平滑） |
| 回传 obs | 力矩电流、转速 rpm、位置 0-360°、绕组温度、错误码 | 位置、速度、负载占空比、**电流 6.5mA/LSB→换算力矩代理**、电压、温度、运动状态 |

- **轮用力矩**：DDSM 是准直驱力矩控制，正是 legged locomotion RL SOTA 首选动作空间（MIT mini-cheetah/ANYmal 同范式）。
- **腿用位置目标**：ST3215 无法指令力矩，但能回读电流→换算力矩做观测。sim 里舵机建模为"位置目标 + PD 内环"，real 用"学习型执行器校正"(Learning Corrective Actuator Dynamics) 补偿死区/回差。

### ADR-4 通信：自定义 UART（否决 micro-ROS）

- **主控链路 Pi↔STM32**：保留自定义 UART 3.3V TTL @921600，帧 `0xA5 + cmd + payload + CRC8/MAXIM + 0x5A`。Pi 侧 `stm32_bridge` ROS2 节点做 serial↔topic 翻译。
- **否决 micro-ROS**：F407 的 192KB RAM 中 micro-ROS 占 50-100KB，叠加 1kHz Mahony+LQR+电机环后余量紧张；开箱实时性不可控（[MDPI 2024 PoDS 论文](https://www.mdpi.com/2079-9296/13/9/1658)）；安全关键链路不应依赖 Pi 上常驻 Agent 进程（Agent 崩溃=链路全断）。Newton Capstone 团队因控制环被压垮最终弃用 micro-ROS 回归 UART。
- **结论**：自定义协议仅占 RAM<8KB，CRC8+心跳完全自控，安全兜底不依赖 Pi/ROS2/DDS。

### ADR-5 STM32：裸机前后台 + HAL/LL 混合

- **裸机前后台 + 1kHz TIM 中断**（控制环在 ISR），主循环跑状态机/舵机/Pi 解析。抖动最小、无上下文切换开销。FreeRTOS 资源够但实时性收益不抵复杂度。
- **HAL/LL 混合**：CubeMX 生成初始化；控制环路径（SPI、TIM、UART DMA、GPIO 方向切换）用 LL/寄存器，延迟低且可预测；非关键外设用 HAL。
- **关键带宽约束**：DDSM315 的 115200 固定波特率是真瓶颈。单方向 10B 帧≈868μs，一发一收≈1.74ms，**2 台同步轮询一圈≈3.7ms，塞不进 1ms 周期**。必须分时到 250Hz(4ms 槽) 交错轮询。

### ADR-6 训练环境：WSL2

- **WSL2 + Ubuntu + CUDA** 跑 MuJoCo MJX + JAX + PyTorch。GPU 直通成熟，`torch.cuda.is_available()=True`。
- **坑规避**：① 项目/conda/数据放 WSL 内 `~/`（ext4），勿放 `/mnt/c`（I/O 极慢）；② `.wslconfig` 配 `memory=`/`processors=`；③ Windows 侧装 NVIDIA WSL 驱动，WSL 内**不装** Linux 驱动；④ Blackwell(RTX50) 早期兼容坑，需最新驱动+PyTorch nightly；⑤ 训练栈基于 MuJoCo Playground / RSL-RL（pip 直装），与 ADR-1 的 MuJoCo MJX 仿真器一致。

**核心设计原则**：
1. **残差 RL 但残差激进**：轮力矩残差、腿位置目标残差，充分利用 DDSM 力矩环 + ST3215 全回传。平衡态残差≈0 是涌现特性而非限制。
2. **Teacher-Student + RMA 三件套**：teacher 用 sim 特权信息训强；student 蒸馏 + RMA latent 在线辨识执行器/载荷变化 → 缩 sim-to-real gap。
3. **sim-to-real 对拍内置**：MJX 主训，原生 MuJoCo 验证，同一 `.xml` 模型。
4. **底层 LQR 永远在环**：RL 挂了 LQR 兜底，满足安全硬约束。

## 二、RL 详设

### 2.0 物理基线（取自《KUAFU》§六 机构分析 / §四 整机参数）

驻留态整机降为经典轮式倒立摆，这是 RL 平衡策略的物理基础：

| 量 | 值 | 来源 |
|---|---|---|
| cart 质量 mc（轮 DDSM315） | 0.698 kg | KUAFU §6.4 |
| pendulum 质量 mp（机身+腿+电子件） | 1.981 kg | KUAFU §6.4 |
| 整机 M | 2.68 kg | KUAFU §4.1 |
| 摆长 Lp（pendulum 质心相对轮轴） | 56.0 mm | KUAFU §6.4 (整机 COM 80.5mm -> pendulum 质心 95.1mm - 轮轴 39.08mm) |
| 轮半径 r | 39.08 mm | KUAFU §3.1 |
| 不稳定自然频率 ωn | 17.17 rad/s（周期 0.37 s） | KUAFU §6.4.1 |
| 摔倒时间常数 | ~65 ms | KUAFU §6.4.1 |
| LQR 状态 | [x, θ, ẋ, θ̇] | KUAFU §6.4.1 |
| LQR 增益 K | [−4.47, −61.18, −5.82, −4.02]（输入 F, N） | KUAFU §6.4.2 |
| 闭环恢复时间常数 | ~1.1 s，ζ≈0.74 | KUAFU §6.4.2 |
| 单轮额定扭矩 | 0.55 Nm（堵转 1.1） | KUAFU §3.1 |
| 舵机连续安全扭矩 | ~1 Nm（堵转 2.94） | KUAFU §3.1 |

> LQR 增益对机身转动惯量敏感（匀质杆近似 vs 盒体实际差 2-3×），首件实测惯量后重标定。RL 观测中显式含角速度与力矩反馈，可吸收此不确定。

### 2.1 观测空间（observation）

分三层：本体感受（proprioceptive，M5 student 用）、特权信息（privileged，仅 teacher 用）、视觉（perceptive，M6 student 用）。

**M5 student 观测（本体感受，~40 维，50 Hz）：**

| 组 | 维度 | 内容 | 说明 |
|---|---|---|---|
| 姿态 | 3 | 机身 roll/pitch/yaw | BMI088 Mahony 输出 |
| 角速度 | 3 | 机身 ωx/ωy/ωz（陀螺） | 1 kHz 原始，降采样 50 Hz |
| 关节状态-轮 | 4 | 左右轮位置、速度 | DDSM 回传 |
| 关节状态-腿 | 8 | 4 舵机位置、速度 | ST3215 回传 |
| **力矩观测-轮** | 2 | 左右轮力矩电流→τ=0.75·I | DDSM 回传，关键 |
| **力矩观测-腿** | 4 | 4 舵机电流→力矩代理 | ST3215 current 6.5mA/LSB，关键 |
| 上一步动作 | 6 | [Δτ_L, Δτ_R, q_hip×4] | 动作平滑性/收敛诊断 |
| 指令 | 3 | [v_cmd, ω_cmd(转向), D₀_cmd(腿高)] | 高层下发的期望速度/转向/腿高 |
| 时钟基 | 2 | sin/cos(2π·t/T_phase) | 步态相位编码（爬阶用） |
| RMA latent | ~5 | 在线辨识的环境参数嵌入 | M5 末引入，见 2.5 |
| 历史 | ×N | 上述部分堆叠 N=4 步 | 部分可观补偿（延迟/接触） |

**teacher 特权观测（额外，仅训练）：**

| 组 | 内容 | 作用 |
|---|---|---|
| 地形高程 | 机身周围高度图（sim 注入） | 提前感知台阶/坑 |
| 摩擦系数 | 接触 μ 真值 | teacher 学会适配 |
| 质量偏置 | 实际 M/COM 偏移真值 | 适应载荷变化 |
| 真实延迟 | actuator/sensor 延迟真值 | 鲁棒性 |
| 外部扰动力 | 施加的扰动力向量 | 抗推 |

**M6 student 视觉观测（额外）：** D435i 深度图经轻量 CNN encoder（~80k 参数）→ 32 维嵌入，残差分支接入本体感受主干。VIO 只用于高层导航/高度估计，**不进低层平衡观测**（VIO 有 30-50ms 延迟，会引入相位滞后）。

### 2.2 动作空间（action）

**混合，6 维，50 Hz：**

| 维 | 对象 | 物理量 | 范围 | 叠加方式 |
|---|---|---|---|---|
| 0 | 左轮 | 力矩残差 Δτ_L | [-1,1]→×τ_max(0.55Nm) | τ_cmd = clip(LQR_out + Δτ_L, ±1.1) |
| 1 | 右轮 | 力矩残差 Δτ_R | [-1,1]→×τ_max | τ_cmd = clip(LQR_out + Δτ_R, ±1.1) |
| 2 | 左前髋 | 位置目标 q_LF | [-1,1]→角度映射 | q_goal = q_center + action·range + accel 平滑 |
| 3 | 右前髋 | 位置目标 q_RF | 同上 | 同上 |
| 4 | 左后髋 | 位置目标 q_LB | 同上 | 同上 |
| 5 | 右后髋 | 位置目标 q_RB | 同上 | 同上 |

- **轮用力矩残差**：利用 DDSM 真力矩环，与 legged SOTA 一致。平衡态 LQR 输出主导，Δτ≈0；扰动/爬阶时 Δτ 修正。
- **腿用位置目标**：适配 ST3215（无纯力矩）。sim 中舵机建模为 position actuator + 内层 PD（增益可调，见 §三）。动作经 goal_speed/accel 平滑，避免阶跃打坏舵机。
- **残差 clip 保护**：力矩残差叠加后硬限幅到堵转 1.1 Nm；位置目标限幅到机械限位 D₀∈[58,207] 对应角度。RL 挂掉 → STM32 忽略 RL 输出，纯 LQR/PD 兜底。

### 2.3 Reward 设计

任务相关（task）+ 风格/平滑（style）+ 安全约束（safety）三组。权重经课程调整。
设计参照 MuJoCo Playground (Go1/T1) 与 ETH legged_gym。所有项 ×scale 后统一乘 CTRL_DT。

**task（鼓励完成任务）：**
- `lin_vel_tracking`：跟踪 v_cmd，exp(-‖v_x - v_cmd‖²/σ²)，σ=0.25，权重 1.0
- `ang_vel_tracking`：跟踪 ω_cmd，同上，权重 0.5
- `d0_avg_tracking`：跟踪 D₀_cmd **左右平均**（仅惩罚对称偏离, 不惩罚左右差 → 允许 roll 调平），权重 0.3
- `orientation`：exp(-α·(gx²+gy²)) 重力向量水平分量，权重 1.0（平衡核心, 无欧拉角万向锁）
- `tilt_cost`：-‖g_xy‖ 线性倾角惩罚（与 orientation 互补, 大倾角恢复激励），权重 0.5
- `roll_leveling`：exp(-roll²/σ²)，σ=0.05，权重 1.0（显式奖励机身水平, 激活腿调平能力）
- `alive`：存活奖励, 对抗过早终止, 权重 0.1。倒下时归零（`alive = 1 - fallen_flag`），防止软终止窗口内蹭存活奖励

**style（平滑、节能、限制 M4）：**
- `extension_cost`：-max(Σ|q_hip| - target_ext, 0)，仅惩罚超出 d0_cmd 的过度伸展（不惩罚命令内伸展, 避免与 d0_avg_tracking 梯度互抵），权重 0.5
- `contact_asymmetry`：-|contact_L - contact_R|（抑制长时间单轮卸载, 限 M4 为短暂），权重 0.3
- `ang_vel_xy`：-(ωx²+ωy²)，权重 0.05（抑制 roll/pitch 抖动）
- `action_rate`：-‖a_t - a_{t-1}‖²，权重 0.01
- `energy`：-Σ|τ·ω|(轮) - Στ²(腿铜损)，权重 0.001
- `torque_limit`：超连续安全扭矩惩罚（腿 1 Nm），权重 0.5

**safety（硬约束）：**
- `fall_penalty`：倒下当步终止惩罚，权重 1.0（配合软终止 200ms 宽限 + alive 门控）
- `soft_termination`：连续倒下≥10 步才终止, 靠 episode 截断 + alive 门控（倒下时 alive=0, fall_penalty=-1, 双重抑制）
- `joint_limit`：超机械限位惩罚
- `leg_overload`：舵机电流超连续安全 → 回锁

> 平衡态 reward 由 `orientation` + `roll_leveling` + `lin_vel_tracking`(v_cmd=0 时=stability) 主导。`d0_avg_tracking` 仅约束左右平均 D₀, 不抑制左右差 → 腿可用于 roll 调平 (一腿高一腿低仍水平)。`contact_asymmetry` 限制 M4 抬轮为短暂, 防单轮持续卸载失稳。

### 2.4 域随机化（Domain Randomization）

teacher 训练时注入，让策略对未知参数鲁棒，缩小 sim-to-real gap：

| 参数 | 范围 | 理由 |
|---|---|---|
| 整机质量 | ±15% (1.88–2.54 kg) | 充电宝/载荷变化 |
| COM 位置 | ±20 mm（X/Y/Z） | 装配/惯量近似误差 |
| 转动惯量 | ×[0.5, 2.0] | 盒体 vs 匀质杆差异 |
| 轮摩擦 μ | [0.3, 1.2] | 地面材质变化 |
| 轮半径 | ±1 mm | 轮胎磨损 |
| DDSM 力矩常数 | ±10% | 电机个体差异 |
| ST3215 PD 增益 | ±30% | 舵机一致性 |
| ST3215 死区/回差 | [0, 2°] | 齿轮间隙 |
| 传感器噪声 | IMU 加速度/陀螺/编码器按实测密度 | 真实噪声 |
| 通信延迟 | actuator [0, 30]ms + sensor [0, 20]ms | UART/总线延迟 |
| 外部扰动 | 随机脉冲力（2 N·s，模拟侧推） | 抗推鲁棒性 |
| 地形 | 平地/坡度±10°/台阶 0-30mm/随机粗糙 | 任务覆盖 |

### 2.5 Teacher-Student 蒸馏 + RMA

**两阶段（RMA, Kumar et al. RSS 2021）：**

**阶段一：Teacher 训练（sim，特权信息）**
- teacher **actor** 输入 = 本体感受(148) + 静态环境外因 latent z(9) = 157 维。本体感受含左右轮接触标志(2 维, M2/M4 涌现关键)。z 编码 episode 级常量的特权环境参数（摩擦 / 质量缩放 / COM 偏移 / 惯量缩放 / 力矩常数 / 死区 / 延迟步数），训练时喂真值，使 base policy 学到 **z 条件化**策略——这是 RMA 能在部署时生效的前提。
- teacher **critic** 输入 = actor obs(157) + 瞬态扰动(3) = 160 维（瞬态 active_push 每步变化，由本体感受在线感知，仅 critic 特权，不进 latent）。
- PPO 训练至收敛。此为非对称 actor-critic：actor 额外见静态 z，critic 见全部特权，价值估计更准而策略可部署。

**阶段二：Student 蒸馏（sim，仅本体感受）**
- student = 主干 MLP [512,512,512]（输入 proprio + z_hat = 157）+ RMA adapter（50 步本体感受历史 → 1D-CNN[32,64,32] → z_hat 9）。
- adapter 以**真值 z 做 MSE 回归**（z 监督），policy head 以 **DAgger** 对齐 teacher 的 action；二者联合训练。
- 部署时 adapter 在线从本体感受历史推断 z_hat，实现自适应（载荷 / 地面 / 延迟变化自动适配），无需任何特权信息。

**M6 视觉扩展：** student 加视觉 encoder 残差分支（冻结本体感受主干权重，只训视觉分支 + 适配层），蒸馏 teacher 的地形特权信息到视觉观测。此为 ETH ANYmal 感知 locomotion 标准做法。

**网络规模（Pi5 ONNX <20ms 控制周期约束）：**
- 主干 MLP：3×512（proprio 148 + z 9 → 512×3 → action 6），参数 ~700k
- adapter：1D-CNN over 50-step 历史 → 9 维 z
- 视觉 encoder（M6）：轻量 CNN ~80k 参数 → 32 维
- Pi5 MLP 推理 ~1.5ms，远低于 50Hz（20ms）控制周期红线

### 2.6 训练管线

```
[WSL2] MJX 仿真 (4096 并行 envs)
   │
   ├─ 阶段0: 纯 LQR baseline 验证（无 RL，确认 sim 物理正确）
   │
    ├─ 阶段1: 残差 RL（M5）
    │    PPO, RSL-RL, teacher actor 条件于 latent z + 域随机化 + 随机推力
    │    curriculum: d_max 由高难度环境存活率双向调节 (≥80%升/≤40%降), per-env 采样 Uniform(0, d_max)
    │               初始 d_max=0.1 即注入 DR+扰动; 平地平衡 -> 坡度 -> 台阶 -> 大扰动
    │    收敛判据（多维）: 恢复时间 < LQR×0.85 (子集) / 扰动承受 > LQR×1.2 /
    │                     速度跟踪误差达标 / 台阶 30mm 成功率 ≥80%
   │
   ├─ 阶段2: Student 蒸馏
   │    adapter z 监督 + policy DAgger/KL
   │    验证: student 无特权信息性能 ≥ teacher×0.9
   │
   ├─ 阶段3: 原生 MuJoCo 对拍（CPU 单环境，可视化确认行为合理）
   │
   ├─ 阶段4: ONNX 导出 + Pi5 延迟测试 (<5ms)
   │
   └─ 阶段5: 实机部署（见 §六）
```

- **算法**：PPO（RSL-RL，legged locomotion 事实标准）。SAC/TD3 备选但不首选（off-policy 对域随机化更敏感）。
- **并行规模**：MJX 4096 envs，单 4090 PPO ~60-650K steps/s。
- **rollout 长度**：每环境 72 步（72×0.02s=1.44s），覆盖完整步态与越障周期，降低 critic bootstrap 偏差，利于长程时序信用分配。
- **课程**：自动课程学习（legged_gym 范式），d_max 双向调节（≥80%升/≤40%降），per-env 采样 Uniform(0, d_max)。

#### 2.6.1 采集实现：逐步采集（RSL-RL + DLPack）

标准 RSL-RL `learn` 对每环境每步发射一次 `env.step` → `alg.act` → `alg.process_env_step`，
rollout 经 DLPack 零拷贝在 JAX(MJX) 与 PyTorch(RSL-RL) 间交换 GPU 张量，训练循环语义与 MJX 原生的逐步采集一致。

曾评估把整段 rollout 塞进单个 `jax.lax.scan`（jax actor + jax `EmpiricalNormalization` 在 scan 内完成，
结果以单次 DLPack 回 torch，复用 PPO 的 `storage` / `compute_returns` / `update`）。
该方案要求 scan 输出为静态形状 `[T, N, ...]`，依赖运行数据长度的筛选（如课程高难度 mask）须放在 Python 端、
jit 内不动态索引，且单 rollout 内固定 `d_max`、结束后才调 `Curriculum.update`。
实测在 RTX 4070 8GB 上，jax 内跑 MLP 慢于 torch 的 cuDNN 路径，逐步采集反而更快、更省显存，
等价性已由 S0 对齐护栏（`rl/verify/s0_parity.py`）覆盖，故主线采用逐步采集，不保留第二套采集实现。

## 三、仿真建模

MuJoCo 对闭链 equality、混合 actuator（motor + position）、sensor 在 MJX 与原生 MuJoCo 均完全支持（[MJX Feature Parity](https://mujoco.readthedocs.io/en/stable/mjx.html)），五杆闭链有现成实例（[Discussion #2662](https://github.com/google-deepmind/mujoco/discussions/2662)）。同一 `.xml` 可用于 MJX 训练与原生 MuJoCo CPU 对拍，经 `mjx.put_model/put_data`/`get_data` 互转。

**地形与 MJX 碰撞约束：** MJX 的碰撞对支持是子集——动态圆柱（轮）与 heightfield / box 的碰撞**未实现**，仅动态圆柱 × 平面可用。因此地形采用 **倾斜平面（斜坡）+ 静态 step box（台阶）** 实现，并将轮 geom 由 `cylinder` 改为 `capsule`（半径不变、加圆端，动态 capsule 与平面/box 碰撞均支持），使斜坡与 30mm 台阶均可与轮碰撞。`difficulty=0` 时平面不倾斜、台阶高度≈0，完全退回已验证的平地行为。

### 3.1 整体 MJCF 结构

```

├─ <compiler> angle=radian, coordinate=local
├─ <option> timestep=0.002 (500Hz 物理子步，RL 50Hz 控制每步 10 子步)
│           solver=Newton, iterations=50 (闭链需多迭代)
├─ <default> <joint> damping/armature/frictionloss; <geom> condim
├─ <worldbody>
│   ├─ <light>/<camera>
│   ├─ 地形 geom (倾斜平面=斜坡 + 4 级 step box=台阶, 训练时按 difficulty 缩放)
│   └─ <body name="chassis">  机架 (自由度 floating base)
│       ├─ 机身 geom (盒体 240×196×50, 9000he PA12 密度)

│   ├─ <position name="q_rf"> 右前髋
│   ├─ <position name="q_lb"> 左后髋
│   └─ <position name="q_rb"> 右后髋
├─ <sensor>
│   ├─ <accelerometer>/<gyro> site=imu (BMI088 仿真)
│   ├─ <jointpos>/<jointvel> 各关节 (编码器)
│   ├─ <actuatorfrc> 各 actuator (力矩/电流回传统计)

```

### 3.2 五杆闭链建模要点

**建模策略**：每腿两根串联肘链（髋A→曲柄a→P1→连杆b→Q，髋B→曲柄a→P2→连杆b→Q），Q 点用 `<equality><connect>` 把两链末端焊在一起形成闭链。两台舵机分别驱动两根曲柄（2-DOF）。

```xml
<!-- 左腿闭链核心（简化） -->
<body name="crank_A_l" pos="-26 0 0">      <!-- 髋点 A(-26,0) -->
  <joint name="hip_A_l" type="hinge" axis="0 1 0"/>  <!-- 舵机驱动轴 -->
  <geom class="link_a" size="..." fromto="0 0 0  ... 93 ..."/>  <!-- 曲柄 a=93 -->
  <body name="link_A_l" pos="...">          <!-- P1 -->
    <joint name="knee_A_l" type="hinge" axis="0 1 0"/>
    <geom class="link_b" fromto="...  ... Q"/>  <!-- 连杆 b=149 到 Q -->
    <site name="Q_A_l" pos="(Q 坐标)"/>      <!-- Q 点 site -->
  </body>
</body>
<!-- 同理髋 B → 曲柄 → P2 → 连杆 → Q_B_l site -->
<equality>
  <connect name="loop_l" site1="Q_A_l" site2="Q_B_l"/>  <!-- 焊 Q = 闭链 -->
</equality>
```

**闭链 solver 稳定性参数**（Discussion #2662 经验）：
- `<option iterations="50">` Newton 求解器，闭链需多迭代防约束漂移
- `<equality><connect solref="0.005 1" solimp="0.99 0.999 0.001"/>` 硬约束
- 闭链关节加 `<joint armature="0.01" frictionloss="0.001"/>` 抑制数值振荡
- timestep=0.002s（500Hz）子步，配合 50Hz 控制每步 10 子步
- **自锁建模**：驻留态 D₀=58 曲柄贴合承重面是非连续接触，难精确建模。reward 中显式惩罚"依赖自锁的姿态"，策略不学依赖自锁；real 中自锁是额外安全裕度。

### 3.3 执行器建模

**轮 — motor（力矩控制，公式经 Discussion #754 核实）：**
```xml
<actuator>
  <motor name="tau_l" joint="wheel_l" gear="1" ctrlrange="-1.1 1.1"/>
  <!-- τ = ctrl × gear = ctrl (Nm), ctrlrange 堵转 1.1Nm -->
</actuator>
```
RL 输出 Δτ，叠加 LQR 输出后 clip 到 [-1.1, 1.1] 写入 ctrl。

**腿 — position（位置控制，模拟 ST3215 内层 PD）：**
```xml
<position name="q_lf" joint="hip_A_l"
          kp="80" kv="2"          <!-- 从舵机标定, 见下 -->
          ctrlrange="(q_min) (q_max)"   <!-- D₀58-207 对应角度 -->
          forcelimit="1.0"/>       <!-- 连续安全扭矩, = ST3215 Torque Limit -->
<!-- τ = kp(ctrl - q) - kv·q̇ , forcelimit 硬截断 -->
```
RL 输出归一化 action∈[-1,1] → 映射到 ctrlrange → 经 goal_accel 平滑。

**舵机 kp/kv 标定**：从 ST3215 可调 P/D 增益寄存器（0x15/0x16）实测映射。初值：堵转 2.94Nm @ 12位编码器，典型阶跃响应，估 kp~80-120 Nm/rad、kv~1-3，首件实测后标定并同步到 sim。

**扭矩-转速曲线（DDSM）**：额定 0.55Nm@200rpm、空载 315rpm。用 `<general>` actuator + 自定义 biasprm 建模反电势项，或简化为 motor + 在 reward/域随机化里随机化扭矩上限（按当前速度查扭矩-转速表限幅）。优先简化版（motor + 速度相关 forcelimit）。

**死区/回差（ST3215）**：在 sim 里对 position actuator 的 ctrl 加死区（|Δ|<阈值不输出）+ 回差（位置滞后），域随机化死区范围 [0, 2°]。或用"学习型执行器校正"——额外训一个小网络预测 sim→real 执行器偏差。

### 3.4 地形 curriculum

- **平地**：`<geom type="plane"/>`，初始课程
- **随机粗糙**：hfield 注入高度噪声（[Issue #1491](https://github.com/google-deepmind/mujoco/issues/1491) 已解决，MJX 支持 hfield），振幅随课程升
- **台阶**：procedural 生成 mesh 台阶（30mm 高，对应 M4 验收），随机间距/高度
- **坡度**：plane 倾斜 ±10°
- **课程**：自动课程学习，d_max 双向调节（≥80%升/≤40%降），per-env 采样 Uniform(0, d_max)，参考 legged_gym terrain curriculum

### 3.5 物理参数注入

从《KUAFU》§三硬件、§五结构设计、§四整机参数提取真实值写入 MJCF：
- 各 geom 密度：结构件统一 9000he（HP MJF PA12）1.01 g/cm³
- 电子件用显式 `<inertial>` 注入质量/惯量（电池 285g、buck 140g、Pi5 85g 等已称重值），整机 COM 离地 80.5mm（注意：cart-pole 摆长 Lp=56.0mm 是 pendulum 质心相对轮轴，不等于整机 COM 高）
- 摩擦：轮-地 μ∈[0.3,1.2] 域随机化
- 接触：轮地 condim=3（含切向摩擦），连杆间 condim=1（仅法向）

### 4.2 ROS2 节点分层

按频率与职责分层，控制环在 STM32，Pi 只做高层决策与推理：

| 节点 | 频率 | 职责 | 关键 QoS |
|---|---|---|---|
| `stm32_bridge` | 200Hz↑收 / 50Hz↓发 | UART↔ROS2 topic 翻译，CRC 校验，心跳 | 见下 |
| `rl_policy_node` | 50Hz | ONNX 推理，输出 action，RMA latent 更新 | in: best_effort depth=1; out: reliable depth=1 |
| `vio_slam_node` | 10Hz | D435i VIO，位姿/高度估计 | reliable depth=5, lifespan=200ms |
| `d435i_node` | 30Hz | 深度图发布 | best_effort depth=2 |
| `decision_node` | 30Hz | 路径规划/目标下发 v_cmd/ω_cmd | reliable depth=1 |
| `diagnostics_node` | 1Hz | 电池/温度/状态汇总 | reliable, liveliness=MANUAL lease=1s |

### 4.3 Topic 与 QoS 设计

```
[STM32] ──UART 921600── [stm32_bridge]
                            │
   ┌────────────────────────┼───────────────────────────┐
   ▼ (200Hz)                ▼ (200Hz)                    ▼ (200Hz)
/imu/data               /joint_states                /robot_state
BEST_EFFORT depth=1      BEST_EFFORT depth=1          BEST_EFFORT depth=1
deadline=10ms            deadline=10ms                deadline=10ms
                                                         │
                                          [rl_policy_node] @50Hz
                                          订阅 /robot_state (BEST_EFFORT)
                                          订阅 /cmd_vel     (RELIABLE, decision 30Hz)
                                          ONNX 推理 <1ms
                                                 │
                                                 ▼ 发布 /joint_cmd (50Hz, RELIABLE depth=1)
                          → stm32_bridge 经 UART 下行 STM32

[vio_slam_node] @10Hz → /odom (reliable depth=5 lifespan=200ms)
[d435i_node] @30Hz → /camera/depth (best_effort depth=2)
[decision_node] @30Hz → /cmd_vel (v_cmd, ω_cmd; reliable depth=1)
                         /leg_height_cmd (D₀_cmd; reliable depth=1)
```

**QoS 原则**：
- 高频传感器（IMU/joint）：`BEST_EFFORT + depth=1 + deadline=周期`，防陈旧数据堆积
- 控制指令：`RELIABLE + depth=1`，丢失重传
- VIO：`reliable + depth=5 + lifespan=200ms`，容忍偶发延迟但不过期
- 用 `deadline` 匹配周期做隐式故障检测（超 deadline = 发布者异常）
- `/robot_state` 是 rl_policy_node 的合成观测 topic（stm32_bridge 聚合 IMU+joint 成观测向量），解耦策略与硬件

### 4.4 仿真↔实机统一接口

关键设计：**rl_policy_node 的输入输出 topic 在 sim 和 real 完全一致**，策略网络无需改代码即可两端跑。

```
        sim (MJX)                          real (Pi5)
┌─────────────────────┐            ┌─────────────────────┐
│ MJX env 包装成 ROS2  │            │ stm32_bridge 真硬件  │
│ 节点，发同名 topic   │            │ 发同名 topic        │
│ /robot_state         │◄── 同一接口 ──►│ /robot_state        │
│ /joint_cmd           │            │ /joint_cmd          │
└─────────────────────┘            └─────────────────────┘
            │                                  │
            └────────── rl_policy_node ────────┘
                 (ONNX, 代码完全相同)
```

这样 sim 训练用的 wrapper（把 MJX obs 写成 /robot_state、把 /joint_cmd 读回 MJX action）与 real 的 stm32_bridge 对策略透明。便于 sim-to-real 对拍与回归测试。

## 五、STM32 底层

### 5.1 时间预算与多速率调度

**1 kHz 周期（TIM 中断 ISR，~70-100μs CPU）：**

| 任务 | 频率 | 单次耗时 | 周期 | 说明 |
|---|---|---|---|---|
| BMI088 SPI 读（DMA） | 1 kHz | ~20-40μs | 1ms | SPI 8-10MHz，DMA 后台 |
| Mahony 融合 | 1 kHz | ~15-30μs | 1ms | F407@168MHz，无 sqrt/trig 密集 |
| UART 收 Pi 指令（DMA+IDLE） | ≥30 Hz 异步 | ~3-5μs CPU | 异步 | DMA 近零 CPU |
| tick 软定时器 | 1 kHz | ~1μs | 1ms | 计数器分发低频任务 |

**250 Hz 电机环（4ms 槽，交错轮询，避开 115200 瓶颈）：**

| 时槽 | 任务 | 耗时 | 说明 |
|---|---|---|---|
| slot 0 (0-1ms) | DDSM 左：发指令→收回包 | ~1.85ms | RS485 半双工，等 TC 切方向 |
| slot 1 (1-2ms) | DDSM 右 | ~1.85ms | |
| slot 2 (2-3ms) | 状态/力矩回传 Pi 缓冲 | <0.1ms | |
| slot 3 (3-4ms) | 空闲/余量 | — | 吸收抖动 |

> 单方向 10B 帧@115200 ≈ 868μs，一发一收 ≈ 1.74ms，**2 台同步轮询塞不进 1ms**，必须分时到 4ms 槽。

**50-100 Hz 舵机环（10-20ms 槽，仅姿态变换/爬阶时）：**
- 4 台 ST3215 @1Mbps TTL，轮询一圈 ~1.2ms
- 平衡态不碰舵机（自锁被动），仅 D₀_cmd 变化或爬阶时激活
- 放主循环软定时槽，不进 1kHz ISR

### 5.2 DDSM315 驱动要点

- **SP3485 方向切换**：必须等 `USART_SR_TC`（最后停止位移出）再拉低 DE/RE，否则截断末字节。优先用 STM32 硬件 RS485 DE 模式（`CR3_DEM`）由外设自动管时序。
- **CRC-8/MAXIM**：poly 反射形式 0x8C，init=0x00，refIn/refOut=true。校验 DATA[0..7]（**不含帧头 0x3E**，易错）。用 256B 查表法。
- **电流环模式**（力矩控制）：命令帧 `ID 64 [I_HI] [I_LO] 00 00 AccTime Brake 00 CRC8`，I=int16 ±32767 ↔ ±8A，τ=0.75·I。RL 残差 Δτ 叠加 LQR 输出后换算电流写入。
- **回传解析**：协议1(0x64驱动)帧含 Mode/TorqueCurrent(int16)/Velocity(rpm)/Position(0-32767=0-360°)/ErrCode。TorqueCurrent 直接用力矩观测。
- **参考**：[waveshareteam/ddsm_example](https://github.com/waveshareteam/ddsm_example) 协议蓝本。

### 5.3 ST3215 驱动要点

- **STS 协议族**（非 SCS）：寄存器地址表用 STS 版（0x38 位置、0x45 电流、0x15/16/17 PID 等）。参考 [python-st3215](https://github.com/Mickael-Roger/python-st3215) 寄存器表。
- **TTL 单线**：STM32 UART HDSEL（`CR3_HDSEL`）单引脚自闭环，或外部方向电路（FE-URT-1）。HDSEL 注意 RX 会收自己发的字节（先 flush）。
- **位置控制**：写 Goal Position(0x2A, 2B) + Goal Speed(0x2E) + Goal Acceleration(0x29)。RL 输出归一化 action→角度→写入。goal_accel 平滑防打坏。
- **力矩观测**：回读 Present Current(0x45, 2B, 6.5mA/LSB)→τ代理，作 RMA adapter 输入。
- **温度保护**：回读 Present Temperature(0x3F)，超阈值（如 65°C）释放扭矩→回承重面锁止。
- **参考**：[ufnalski/scs15_servo_half_duplex](https://github.com/ufnalski/scs15_servo_half_duplex_l432kc) 硬件层模板。

### 5.4 基层控制接口规格（STM32 ↔ Pi5, 三轴兜底）

基层控制律在 STM32 1kHz ISR 执行, 与 sim `kuafu_mjx_env._lqr_balance` 同源 (增益真源 `kuafu_physics.py`)。RL 残差由 Pi5 @50Hz 下行, 叠加在基层之上。

**增益表（`kuafu_physics.py` 单一真源, 固件同源镜像）：**

| 轴 | 控制律 | 增益 | 说明 |
|---|---|---|---|
| pitch | LQR `F = -K·[0,θ,ẋ,θ̇]`, τ_p = F·R/2 | K=[-4.47,-61.18,-5.82,-4.02] | 已验证 0.1s 恢复 5° |
| yaw | 条件阻尼 `Δτ = -Kd·ωz` (仅 \|ωz\|<0.3) | Kd=0.05, thresh=0.3 rad/s | 大 ωz 关闭防 pitch 耦合 |
| roll | 腿 PD `ΔD0 = -Kp·roll - Kd·ωx` | Kp=0.15, Kd=0.03 | 0.1s 恢复 3°, 左右 D0 差 |

**Pi5 → STM32 @50Hz 下行帧（RL 残差 + 命令）：**
```
0xA5 [cmd=0x10] [v_cmd(2)] [ω_cmd(2)] [d0_cmd(2)]
      [Δτ_L(2)] [Δτ_R(2)] [Δq_A_l(2)] [Δq_A_r(2)] [Δq_B_l(2)] [Δq_B_r(2)] CRC8 0x5A
```
- v_cmd/ω_cmd/d0_cmd: int16, 量程见 V/W/D0_CMD_RANGE
- Δτ_L/R: int16, ±TAU_WHEEL_RATED(0.55Nm) 归一化
- Δq: int16, ±HIP_STROKE(1.52rad) 归一化

**STM32 → Pi5 @200Hz 上行帧（观测）：**
```
0xA5 [cmd=0x81] [roll(2)] [pitch(2)] [yaw(2)] [ωx(2)] [ωy(2)] [ωz(2)]
      [vx(2)] [wheel_L_pos(2)] [wheel_R_pos(2)] [wheel_L_vel(2)] [wheel_R_vel(2)]
      [τ_L(2)] [τ_R(2)] [hip_A_l(2)] [hip_A_r(2)] [hip_B_l(2)] [hip_B_r(2)]
      [contact_L(1)] [contact_r(1)] CRC8 0x5A
```
- contact_L/R: uint8, 轮接地标志 (1=接地, 0=离地), 由 STM32 据轮力矩+转速估计

**STM32 1kHz 基层 C 伪代码：**
```c
// 每步: 读 IMU(Mahony) → roll,pitch,yaw,ωx,ωy,ωz; 读轮编码器 → ẋ
float F = -(K[1]*pitch + K[2]*xdot + K[3]*thetadot);  // pitch LQR
float tau_p = F * R / 2.0f;
float tau_diff = (fabsf(yaw_rate) < YAW_DAMP_THRESH)
               ? clampf(-YAW_KD * yaw_rate, -TAU_RATED, TAU_RATED) : 0.0f;
// 轮: 基层 + RL 残差 (RL 挂时残差=0)
ddsm_set_torque(LEFT,  tau_p + tau_diff + delta_tau_L);
ddsm_set_torque(RIGHT, tau_p - tau_diff + delta_tau_R);
// 腿: D0 对称 + roll PD + RL 残差
float dd0 = -(ROLL_KP * roll + ROLL_KD * omega_x);
float dd0_rad = dd0 / (D0_MAX - D0_MIN) * 2 * HIP_STROKE;
st3215_set_pos(A_L, d0_A_base + dd0_rad/2 + delta_q_A_l);
st3215_set_pos(A_R, d0_A_base - dd0_rad/2 + delta_q_A_r);
st3215_set_pos(B_L, d0_B_base + dd0_rad/2 + delta_q_B_l);
st3215_set_pos(B_R, d0_B_base - dd0_rad/2 + delta_q_B_r);
```

### 5.5 安全状态机

```
            ┌──────────────────────────────────────────────┐
            │                                              │
   上电 ──► INIT ──► STAND(驻留自锁, LQR轮平衡, 腿不动)
                          │
            ┌─────────────┼──────────────┐
            │心跳正常      │RL激活         │异常
            ▼             ▼              ▼
       ACTIVE(残差RL)  CLIMB(姿态变换)  FAULT
       LQR+Δτ@250Hz    舵机@50-100Hz   ↓
            │             │           判据:
            │             │           - 心跳超时>200ms
            │             │           - |pitch|>45° 摔倒
            │             │           - 舵机>65°C
            │             │           - LiPo<3.0V/节
            │             │           - IWDG 复位
            └─────────────┴──────────► SAFE_LOCKDOWN
                                      腿回承重面锁止
                                      电机切PD位置保持
                                      缓减速至停
                                      发故障码给Pi
                                      喂狗保持safe state
```

**保护实现**：
- **心跳超时**：Pi 下行帧含递增序号，STM32 200ms 未收 → FAULT。自定义协议下完全自控，不依赖 Pi/ROS2。
- **IWDG**：独立看门狗（LSI 时钟），仅当状态机推进到 SAFE_LOCKDOWN 才喂狗；正常时 IWDG 超时复位 = 兜底。
- **WWDG**：窗口看门狗 EWI 中断，复位前做一次优雅关断（电机断能、舵机释放）。
- **LiPo 过放**：ADC 采分压，单节 <3.3V 报警 → <3.0V 切 SAFE_LOCKDOWN。
- **供电独立性**：STM32 经 MP1584 由主电池独立供电，**禁经 Pi USB 反向取电**（板载 USB VBUS 直连 5V 轨，Pi 崩溃会拖垮 STM32，§七保护失效）。

## 六、硬件接口与部署链路

### 6.1 Pi↔STM32 UART 帧协议（自定义，CRC8/MAXIM）

```
下行 (Pi→STM32, 50Hz):   0xA5 [CMD] [SEQ] [PAYLOAD N B] [CRC8] 0x5A
上行 (STM32→Pi, 200Hz):   0xA5 [CMD] [SEQ] [PAYLOAD N B] [CRC8] 0x5A
```
- CRC-8/MAXIM poly 0x8C, init 0x00, refIn/refOut true（与 DDSM 同，复用查表）
- 电平：Pi TX 3.3V 直连 STM32 RX（两边同为 3.3V 域，无需分压）；STM32 TX 3.3V 直 Pi RX（Pi 5V 容忍）。仅 GND 必须共接，电平参考才一致。注：若 STM32 板载 PC7 外部已布分压网络，需先确认，避免叠加导致高电平不足。

**下行 CMD（Pi→STM32）：**

| CMD | 含义 | Payload |
|---|---|---|
| 0x01 | 心跳 + 模式指令 | [mode(1)] [v_cmd(2)] [ω_cmd(2)] [D₀_cmd(2)] |
| 0x02 | RL action | [Δτ_L(2 int16)] [Δτ_R(2)] [q_LF(2)] [q_RF(2)] [q_LB(2)] [q_RB(2)] |
| 0x03 | 参数(增益/LQR K/模型路径) | [key(1)] [value(4)] |
| 0x04 | RMA latent 注入(可选) | [z(5×4 float32)] |

**上行 CMD（STM32→Pi，200Hz）：**

| CMD | 含义 | Payload |
|---|---|---|
| 0x81 | 状态包 | [roll(2)][pitch(2)][yaw(2)][ωx(2)][ωy(2)][ωz(2)] |
| 0x82 | 关节状态 | [wheel_L_pos(2)][vel(2)][τ(2)][wheel_R...][hip×4 pos/vel/cur] |
| 0x83 | 诊断 | [batt(2)][temp_max(1)][err_flags(1)] |
| 0x8F | 故障码 | [code(1)] |

> 200Hz 上行保证 rl_policy_node 的 50Hz 观测新鲜（最近一次状态 <5ms），且 deadline QoS 兜底。

### 6.2 ONNX 推理节点

```python
# rl_policy_node (rclpy, 50Hz)
class RLPolicyNode(Node):
    def __init__(self):
        self.obs_buf = ObservationBuffer(history=4)
        self.adapter = RMAAdapter()  # 在线 z 推断
        self.sess = ort.InferenceSession("policy.onnx", sess_opt)
        sess_opt.graph_optimization_level = ort.ORT_ENABLE_ALL
        sess_opt.intra_op_num_threads = 2
        self.create_subscription(RobotState, '/robot_state', self.on_state, qos_be_depth1)
        self.create_subscription(CmdVel, '/cmd_vel', self.on_cmd, qos_reliable)
        self.pub = self.create_publisher(JointCmd, '/joint_cmd', qos_reliable_depth1)
        self.timer = self.create_timer(0.02, self.infer)  # 50Hz

    def on_state(self, msg):
        self.obs_buf.push(msg)   # 缓存历史

    def infer(self):
        obs = self.obs_buf.pack()           # ~40 维 + history
        z = self.adapter.update(obs)        # RMA 在线 latent
        action = self.sess.run(None, {'obs': obs, 'z': z})[0]
        self.pub.publish(action_to_joint_cmd(action))
```
- 推理 <1ms（Pi5 benchmark，MLP 量级），50Hz 周期 20ms 余量充足
- RMA adapter 用滑动窗口（最近 50 步）推断环境参数 z，实现在线自适应
- 模型热加载：参数服务 `rosservice` 触发重载 ONNX，不停机切策略
